Welcome to HTML5 Semantics

This page demonstrates how to use HTML5 semantic elements to create a well-structured, accessible webpage that follows SEO best practices.

Beginner Tip

Semantic HTML uses elements that clearly describe their meaning to both browsers and developers.

Why Semantic HTML Matters

Semantic HTML helps search engines understand your content, improves accessibility for screen readers, and makes your code easier to maintain.

Common Semantic Elements

  • <header> - Introductory content
  • <nav> - Navigation links
  • <main> - Main content of the document
  • <article> - Self-contained composition
  • <section> - Thematic grouping of content
  • <aside> - Content indirectly related to main content
  • <footer> - Footer for its nearest section
<article>
    <header>
        <h2>Article Title</h2>
        <p>Posted on January 1, 2023</p>
    </header>
    <section>
        <p>Article content goes here...</p>
    </section>
    <footer>
        <p>Article footer content</p>
    </footer>
</article>

Accessibility Basics

Web accessibility ensures that people with disabilities can perceive, understand, navigate, and interact with websites.

Simple Accessibility Tips

  • Use semantic HTML elements
  • Provide alt text for images
  • Ensure sufficient color contrast
  • Use descriptive link text
  • Make forms accessible with proper labels

Did You Know?

Using proper heading structure (h1 > h2 > h3, etc.) is one of the most important aspects of web accessibility.

SEO Fundamentals

Search Engine Optimization helps your website appear in search results. Semantic HTML forms the foundation of good SEO.

HTML Elements That Help SEO

  • Title tags (<title>)
  • Meta description tags
  • Heading tags (<h1> to <h6>)
  • Semantic structure elements
  • Image alt attributes

  • aria-label - for element labels

  • tabindex - controls tab order
